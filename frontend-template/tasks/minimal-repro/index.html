WebClient expected_output_recipients Bug Repro

What this demonstrates

When a note script calls output_note::create(), the kernel needs the output note's script pre-loaded in the DataStore. The Rust client's TransactionRequestBuilder::expected_output_recipients() handles this correctly. The WebClient's withExpectedOutputRecipients() does not — the kernel fails with NOTE_BEFORE_CREATED event error (ID: 1276704095326615571).

Prerequisites

  1. A Miden testnet account with the battleship component deployed
  2. A shot note published to that account (created by the opponent)
  3. The result_note.masp package file available at /packages/result_note.masp

How to reproduce (conceptual)

// 1. Initialize WebClient
const client = new WebClient();
await client.initialize(/* rpcUrl, prover */);

// 2. Load the output note script from .masp
const maspBytes = await fetch("/packages/result_note.masp").then(r => r.arrayBuffer());
const pkg = Package.deserialize(new Uint8Array(maspBytes));
const resultScript = NoteScript.fromPackage(pkg);

// 3. Get the input note (shot note that calls output_note::create())
const noteRecord = await client.getInputNote(noteId);
const note = noteRecord.toNote();

// 4. Build transaction request with expected output recipients
const noteAndArgs = new NoteAndArgs(note);
const noteAndArgsArray = new NoteAndArgsArray([noteAndArgs]);

const dummyInputs = new NoteInputs(new FeltArray());
const recipient = new NoteRecipient(serialNum, resultScript, dummyInputs);
const recipientArray = new NoteRecipientArray([recipient]);

const txRequest = new TransactionRequestBuilder()
  .withInputNotes(noteAndArgsArray)
  .withExpectedOutputRecipients(recipientArray)  // <-- This should pre-load the script
  .build();

// 5. Execute — FAILS with NOTE_BEFORE_CREATED
const txResult = await client.executeTransaction(accountId, txRequest);
// Error: "error during processing of event with ID: 1276704095326615571"

Expected behavior

executeTransaction succeeds (as it does with the native Rust client). The kernel finds the result note's script via the DataStore and creates the output note.

Actual behavior

Kernel fails at the NOTE_BEFORE_CREATED event (ID: 1276704095326615571). The script is not found in the DataStore despite being provided via withExpectedOutputRecipients().

Evidence that script roots match

RESULT_SCRIPT_ROOT from config: [15171288892435243614, 10295758693372466955, 709059778587687919, 9826546822224790371]
Roots match: true

Shot note's result_script_root (inputs[7..10]): [15171288892435243614, 10295758693372466955, 709059778587687919, 9826546822224790371]
Shot inputs match config: true

Working Rust equivalent

let request = TransactionRequestBuilder::new()
    .input_notes([(shot_note, None)])
    .expected_output_recipients(vec![result_recipient])
    .build()?;
client.submit_new_transaction(account_id, request).await?;
// SUCCESS — kernel creates the output note
